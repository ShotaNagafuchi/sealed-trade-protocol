# Sealed Trade Protocol: 情報封印型交渉による P2P 相対取引基盤

**Shota Nagafuchi**

sealed-trade@proton.me

## 概要

相対取引における私的情報の二重使用を防止するプロトコルを提案する。従来の非公開市場では、交渉行為そのものが私的評価額を漏洩させる。買い手の支払意思額、売り手の売却緊急度 —— これらの情報は取引成立のために一度使用され、さらに条件悪化のためにもう一度使用される。これは Bitcoin の二重支払い問題の情報版である。同一の私的情報が、取引の促進と条件の搾取に二重に使用される。

Sealed Trade は、Trusted Execution Environment（TEE）内で動作する AI エージェント間に交渉を限定することでこの問題を解決する。各エージェントはプリンシパル（依頼者）が暗号署名したパラメータに束縛され、A2A（Agent-to-Agent）プロトコルで交渉し、完了後にすべての交渉状態を証明可能な形で破壊する。エンクレーブの境界を越えるのは最終的な合意（または「不成立」の信号）のみである。

決済はボンド付きスマートコントラクトによりオンチェーンで行われる。取引量ベースの半減期を持つ Bitcoin 型マイニング機構により、中央集権的なマーケットメイキングなしに初期採用を促進する。

## 1. はじめに

Bitcoin は電子現金の二重支払い問題を解決した。信頼できる第三者なしに、同一のデジタル通貨を二度支払うことはできない [1]。分散型タイムスタンプサーバーと Proof-of-Work コンセンサスにより、P2P 電子決済から銀行を排除した。

我々は相対取引における類似の問題を特定する：**私的情報の二重使用**。二者間で取引を交渉する際、各当事者の私的情報 —— 留保価格、緊急度、代替選択肢 —— は交渉過程で必然的に伝達される。一度伝達された情報は、公正な取引が生み出す以上の余剰を相手方が搾取するために使用される。

特許ライセンス交渉を考えよう。ライセンシーは支払える最大額を知っている。ライセンサーは受け入れる最小額を知っている。理想的な交渉では、いずれの当事者も相手方の境界を知ることなく、合意可能領域（ZOPA）内の価格で合意する。しかし実際には、あらゆるオファーが情報を漏洩させる。10万ドルの初回提示は、少なくとも10万ドルの支払意思を示す。50万ドルのカウンターオファーは、最大でも50万ドルでの受諾意思を示す。反復的なラウンドを通じて、各当事者の私的評価額は段階的に開示され、搾取される。

### 1.1 貢献

1. **情報の二重使用問題**を、発見時の漏洩・交渉時の搾取・決済後の残存という3層の問題として定式化
2. TEE 限定型 AI エージェント交渉と状態破壊の暗号証明により3層すべてに対処するプロトコルの提示
3. ベンチャーキャピタルや集中型マーケットメイキングなしにネットワーク効果を起動する、Bitcoin 型半減期付き**取引量ベースマイニング機構**の設計
4. スマートコントラクト5本、Python 経済シミュレーション、103件の自動テストを含む動作実装の提供

## 2. 情報の二重使用問題

### 2.1 問題の定式化

相対取引において、私的情報は相反する二つの目的に使用される：

1. **合意の到達**：双方が相互に受容可能な条件を見つけるために、選好を伝達しなければならない
2. **個別余剰の最大化**：各当事者は相手方による価値搾取を防ぐため、選好を秘匿したい

根本的な緊張は、(1) に必要な情報が (2) で悪用される情報と同一であることにある。我々はこれを**情報の二重使用**と呼ぶ：取引を促進するために一度使用された私的情報が、条件を悪化させるために再度使用される。

### 2.2 三つの層

この問題は取引ライフサイクルの3つの異なる時点で顕在化する：

**第1層：発見時の漏洩。** 資産への関心表明が需要を暴露する。特許権者に連絡した買い手は、その特許が戦略的価値を持つことを暴露している。交渉開始前に売り手は価格を上方調整できる。

**第2層：交渉時の搾取。** 各オファーとカウンターオファーがシグナルとなる。アンカリング効果、応答タイミング、譲歩パターン、交渉継続の選択自体が、私的評価額に関する情報を漏洩させる。

**第3層：決済後の残存。** 取引成立後、仲介者（ブローカー、アドバイザー、プラットフォーム）は双方の留保価格と交渉行動の完全な知識を保持する。この情報は将来の取引、他の顧客との共有、関連取引での活用に使用されうる。

### 2.3 既存手法の限界

**信頼できる仲介者**（ブローカー、投資銀行）は守秘義務を約束するが、技術的な強制手段を持たない。彼らのビジネスモデルは情報の非対称性に依存している。グローバル M&A アドバイザリー市場は取引額の1-5%を請求する [3]。

**暗号学的手法**（MPC、FHE、ZKP）は特定の計算を保護できるが、複雑な相対取引に必要な自由形式の自然言語交渉をサポートできない。MPC は事前定義された回路を要求する。FHE は10,000-100,000倍の計算オーバーヘッドを課す [4]。ZKP は計算の正しさを証明するが、任意のコンテンツを封印できない。

**ダークプール**（Renegade [5]、Penumbra [6]）は代替可能トークンスワップの情報漏洩を解決する。しかし相対取引は、単純な価格マッチングではなく多次元交渉を必要とする非代替性の複雑な資産（特許、企業、不動産）を扱う。

## 3. プロトコル設計

### 3.1 概観

```
第3層: 業種別アダプター        （IP ライセンス、不動産、M&A —— 将来）
第2層: 封印型相対取引          （本プロトコル）
第1層: TEE 基盤               （Intel TDX / AMD SEV-SNP）
第0層: 決済                    （L2 上の EVM スマートコントラクト）
```

### 3.2 取引ライフサイクル

取引は6つの状態を遷移する：

```
出品 → マッチ → 交渉中 → 合意 → 決済完了
                                 ↗
              （任意の状態）→ キャンセル
```

**出品。** 売り手がハッシュ化した資産記述と最大取引額を公開し、発見ボンド（取引額の1%、$1-$1,000）を供託する。

**マッチ。** 買い手が関心を表明し、同額の発見ボンドを供託する。

**交渉中。** 双方が交渉ボンド（3%、$5-$5,000）にエスカレート。各当事者が AI エージェントの許容範囲を定義するパラメータセットに署名する。署名済みパラメータが TEE エンクレーブにロードされ、エージェントが A2A プロトコルで交渉する。

**合意。** エージェントが合意に達した場合、双方が実行ボンド（10%、$10-$50,000）にエスカレート。合意ハッシュと TEE 証明がオンチェーンに記録される。

**決済完了。** 取引額（USDC 建て）が買い手から売り手に移転され、0.3%のプロトコル手数料が控除される。ボンドが返還され、マイニング報酬が分配される。

**キャンセル。** 合意前のいずれの時点でも、いずれの当事者もキャンセル可能。交渉開始前はペナルティなしでボンド返還。交渉中のキャンセルでは、キャンセル側のボンドがスラッシュされる（50%が相手方、50%が保険プールへ）。

### 3.3 TEE 基盤

エージェントは Trusted Execution Environment（Intel TDX または AMD SEV-SNP）内で動作する。TEE は3つの特性を提供する：

1. **メモリ隔離。** エンクレーブのメモリはハードウェアレベルで暗号化される。いかなるプロセス、OS、ハイパーバイザー、物理的アクセスも読み取れない。
2. **リモート証明。** TEE は特定のコードが真正なエンクレーブで実行されていることの暗号的証明を生成する。
3. **破壊の証明。** 交渉完了時に、TEE はすべてのエンクレーブメモリがゼロ化されたことを証明する。カーネルレベルの操作であり、フォレンジック復元は不可能である。

#### 信頼仮定

TEE はハードウェアベンダーがメモリ隔離を正しく実装していることへの信頼を要求する。この信頼仮定は：

- ECDSA が離散対数問題の困難性を仮定するのと**同等**
- 10年以上の運用実績で**経験的に検証済み**
- 保険プールにより**経済的に制限される**（違反時の期待損失 ≈ 10⁻⁷ × 取引額）

## 4. 経済モデル

### 4.1 トークン供給

| 割当 | 数量 | 比率 |
|------|------|------|
| マイニング | 95,000,000 SEAL | 95% |
| トレジャリー | 5,000,000 SEAL | 5% |
| **合計** | **100,000,000 SEAL** | **100%** |

全トークンはデプロイ時に発行。インフレーションなし。追加発行機能なし。

### 4.2 取引量ベースの半減期

マイニング報酬は累積取引量の増加に伴い減少する。Bitcoin 型の半減スケジュールを、ブロック高ではなく取引量でインデックスする。

累積取引量 $V$ に対し、期間 $n$ は区間 $V_{n-1} < V \leq V_n$ を占める。ここで $V_n = H \cdot 2^n$、$H = \$10{,}000$。

期間 $n$ の SEAL 割当は $A_n = A_0 / 2^n$（$A_0 = 47{,}500{,}000$ SEAL）。

| 期間 | 累積閾値 | SEAL 割当 | マイニングレート |
|------|---------|----------|----------------|
| 0 | $10,000 | 47,500,000 | 4,750 SEAL/$1 |
| 1 | $20,000 | 23,750,000 | 2,375 SEAL/$1 |
| 2 | $40,000 | 11,875,000 | 593.75 SEAL/$1 |
| 3 | $80,000 | 5,937,500 | 148.44 SEAL/$1 |
| ... | ... | ... | ... |
| 21 | ~$21B | ~23 | ~0 SEAL/$1 |

全供給をマイニングするための累積取引量：約210億ドル。

### 4.3 貢献スコアと報酬分配

取引額 $d$ で決済が完了すると：

$$s_{\text{buyer}} = s_{\text{seller}} = d / 2$$

各期間 $n$ における貢献者の報酬：

$$R_i^{(n)} = A_n \cdot \frac{s_i^{(n)}}{\sum_j s_j^{(n)}}$$

期間割当 $A_n$ の全額が参加者に分配される。

### 4.4 反射的成長メカニズム

プロトコルは取引量とトークン価値の間に正のフィードバックループを示す：

```
取引増加 → 手数料増加 + マイニング → トークン需要増加
                                          ↓
トークン価値上昇 → 取引インセンティブ増加 → 取引増加
```

業種横断的な信頼蓄積がこのループを強化する。IP ライセンスで蓄積した貢献スコアは、不動産取引にも持ち越される。

### 4.5 ジェネシスブートストラップ

プロトコルは $70（¥10,000）の自己取引1件から起動可能。期間0のマイニングレート4,750 SEAL/ドルにより、このジェネシス取引で約332,500 SEAL（供給量の0.33%）がマイニングされる。

これは Satoshi が2009年1月に1台の PC でブロック0をマイニングしたことと意図的に類似させている。

## 5. セキュリティモデル

プロトコルは3つの独立した防御層を提供する：

**暗号学的。** ドメイン分離付き EIP-712 型付き署名によりクロスチェーンリプレイを防止。署名の展性は EIP-2 s 値範囲チェックにより拒否。

**経済的。** ボンドが誠実な行動への経済的インセンティブを形成。保険プールが TEE 違反に対する集合的保護を提供。

**運用的。** スマートコントラクトが状態機械を強制。CEI パターンと ReentrancyGuard がリエントランシー攻撃を排除。Ownable2Step が所有権の安全な移転を保証。

### 5.1 限界

- **TEE ハードウェア信頼。** プロトコルの秘匿性保証は TEE ベンダーの完全性に依存する。
- **オラクル依存。** 取引額は自己申告であり、インフレ報告により追加トークンをマイニング可能。ボンド要件がこの攻撃コストを比例的に生成。
- **未監査。** 実装はプロフェッショナル監査を受けていない。実資金での使用は監査完了まで禁止。

## 6. 実装

5つの Solidity スマートコントラクト（Arbitrum 対応）と Python 経済シミュレーションとして実装。

| コントラクト | 目的 |
|-------------|------|
| SealToken | ERC-20（100M 固定供給、ERC-20 Permit） |
| SealedTrade | 取引ライフサイクル管理（EIP-712 署名） |
| BondVault | 3段階ボンドエスクロー（CEI + ReentrancyGuard） |
| ContributionLedger | 半減期付きマイニング（取引量ベース） |
| Treasury | 手数料金庫 + 保険プール（Ownable2Step） |

テストスイート：Solidity 53件 + Python 50件 = 103件の自動テスト。

モンテカルロシミュレーションにより、全テスト市場シナリオにおいてジニ係数 < 0.2 の公平な報酬分配を確認。

## 7. 結論

相対取引における情報の二重使用問題は、電子現金における二重支払い問題の類似物である。Bitcoin が信頼できる第三者なしの P2P 電子決済を可能にしたように、Sealed Trade は私的情報の二重使用能力を排除することで、信頼できる第三者なしの相対交渉を可能にする。

実装はオープンソースであり、103件の自動テストが経済モデルとスマートコントラクトロジックの正確性を検証している。
